## 1. 這個專案是什麼

**bridge-analyst 是 `district-analyst` 方法論的第二個應用。**

任務來自 時代力量 AI 職位面試的小作業：針對 **淡江大橋** 做議題探索
+ 輿情分析，並在報告書中：
1. 說明 AI 設計的思路和結構
2. 對時代力量的主張和下一步行動提出建議

**我們的策略不是「恰好會做淡江大橋」，是「我們有一套框架，淡江大橋是它第二次跑起來」。**
這個重複本身就是賣點 — 它證明這套方法是通用的。

## 2. 與 district-analyst 的同構關係

| district-analyst (選區分析)         | bridge-analyst (議題分析)              |
|--------------------------------------|----------------------------------------|
| 選區 = 分析單位                      | 議題 = 分析單位                        |
| SQL 抽事實（中選會資料）             | scraper 抽事實（輿論、公部門文件）     |
| Agent = 推理單元                     | Agent = 推理單元                       |
| Prompt 是 first-class artifact       | Prompt 是 first-class artifact         |
| 每次 run 留 trace                    | 每次 run 留 trace                      |
| 辯證推理（假設→反證→判斷）           | 辯證推理（假設→反證→判斷）             |
| 差異化輸出 = 品質檢核                | 差異化輸出 = 品質檢核                  |
| 兩選區對照（竹北 vs 湖口）           | 多個 framing 對照（A/B/C/D）           |

**兩邊的程式碼要長得像。** 如果 bridge-analyst 的 `BaseAgent` 跟
district-analyst 的不一致，是 bug 不是 feature — 未來應該抽成共用
package，但面試這個時程先複製、保持一致性。

## 3. 淡江大橋 — 核心事實 (2026-04-22 cutoff)

> 以下是高密度事實包。完整版看 `data/facts/*.md`。

### 3.1 基本資料
- **位置**：連接 新北淡水 ↔ 八里
- **定位**：台 **61 線** 快速公路系統（`<- 這個字串要記住，是爭議關鍵`）
- **設計**：Zaha Hadid Architects，世界最大單塔不對稱斜張橋
- **全長**：約 8.5 公里（含連絡道路）
- **總經費**：**230.38 億 NTD**（後述三度追加）
- **通車日**：**2026-05-12**（5/9 舉行通車典禮）

### 3.2 關鍵時間軸
- 2010-04: 規劃預算 141 億
- 2017–2018: 主橋段第三標 **連續流標 7 次**，停工 2 年+
- 2018-11: 追加 58 億 → 212.3 億（蘇貞昌/賴清德核定）
- 2019-03-13: 第三標開工
- 2023-07: 再追加 18.08 億（俄烏戰爭原物料）→ **230.38 億**
- 2025-07: 主橋塔完工
- 2025-09-16: 主體工程合龍
- 2025 年底: 整體完工
- 2026-04-18/19: 通車前路跑 + 自行車活動（7000 人參加；輕軌擠爆）
- **2026-04-19 以降: 機車道 2.5m 爭議爆發** ← 這是你做這份報告最重要的 trigger 事件
- 2026-04-20: 運安會宣布納入研討
- **2026-05-12: 正式通車**

### 3.3 機車道 2.5m 爭議（當前最熱議題）

**事實：**
- 設計：實體分隔、單向單車道、**2.5 公尺寬**
- 公路局主張：符合《公路路線設計規範》規定（快速公路車道最小 2.5m）
- 實際可行駛空間：1.5–1.6m（車道寬度 - 路肩 - 分隔設施）
- 設計理由（流出版本）：「怕騎士看風景不專心、怕超車」— 被路權團體痛批

**各方立場：**
| 角色 | 立場 | 備註 |
|------|------|------|
| 公路局 | 防禦 — 符合規範、技術上無法加寬（強風） | 官方口徑 |
| 運安會 | 納入「運輸安全研討會」討論 | 中立 |
| 陳世軒（新北市議員，**民眾黨 TPP**）| 要求降編為「一般省道」、汽機車混流 | **TPP 已搶先佔位** |
| 洪孟楷（立委，**國民黨 KMT**）| 向公路局反映民眾疑慮 | 中立偏安全立場 |
| 台灣機車路權促進會 | 痛批設計理由、「鬼扯」 | **已動員，尋找政治盟友** |
| **時代力量 NPP** | **目前沒有發聲** | ← 這是空白，也是機會 |

### 3.4 交通衝擊預估
- 預計分擔 台 2 線竹圍段 **17%**、關渡大橋 **30%** 車流
- 連動工程：沙崙路隧道、**淡北道路**（2029-06 完工目標）
- 路跑當天 **淡海輕軌擠爆** — 配套脆弱的信號

## 4. 時代力量（NPP）定位分析

### 4.1 當前困境（報告裡要誠實面對）
- 2024 大選後進入「後黃國昌時代」，黨組織收縮
- 淡水/八里 **沒有強地方組織**
- 在此議題上 **尚未發聲** — 落後 TPP、KMT 各自表述
- 但 NPP 的 DNA（進步價值、程序正義、反黑箱）天然適合工程監督議題

### 4.2 TPP 已經佔了什麼（避免複製）
- **陳世軒**（TPP 新北議員）已經佔了「機車路權 + 道路平權」角度
- 主張「解編省道、汽機車混流」是具體政策訴求
- NPP **不能直接複製** — 複製就是 me-too，政治上是送分題給 TPP

### 4.3 NPP 的四個候選 framing（詳見 `data/facts/npp_framing_candidates.md`）

**A. 規範本身就是問題**
> 「2.5m 合規範」本身就揭露一個事實 — 規範是 40 年前制定的，
> 沒人覺得該改。NPP 主張：推動《公路路線設計規範》修法，
> 把「用路人體感安全」納入規範。
- 優勢：超越個案、制度性修法，是 NPP 強項
- 風險：技術門檻高，不好做成社群短訊

**B. 通橋之後的生活政治**
> 不糾纏機車道（讓 TPP 去打），而是打 **「淡江大橋通車只是第一天」**
> — 輕軌擠爆、八里配套、淡海炒作、北海岸承載、環境監測 —
> 整合式都市議題的長期監督。
- 優勢：正面議程、長期視角、延伸 NPP 的「世代正義」敘事
- 風險：訴求抽象，不易短期動員

**C. 230 億的機會成本**
> 230 億 = 流標 7 次 + 追加 90 億的代價。如果投在公共運輸，
> 淡海輕軌路網能擴張多少倍？公車快線能開幾條？問的不是
> 「該不該建橋」，是 **「這種決策模式合理嗎」**。
- 優勢：結合世代運輸正義 + 程序監督，NPP 雙打
- 風險：橋已經蓋好，反向敘事技術上困難

**D. 程序正當性 — 為下一個橋做準備**
> 淡江大橋的經驗教訓要如何制度化？下一個重大工程（淡北道路、
> 其他大橋）如何不要再重演流標 7 次、追加 90 億、安全諮詢
> 黑箱？NPP 主張：**立法建立重大工程的公民監督機制**。
- 優勢：NPP 歷史主張（監督、透明）完美連接
- 風險：不搶熱度，但是有延續性

**我的初判：B + D 組合** 是 NPP 最適合的路。
給 A/C 留在報告裡討論為什麼排除，是決策 legibility 的一部分。

## 5. 系統架構

```
      ┌─────────────────────────┐
      │ scripts/init_db.py       │
      └──────────┬───────────────┘
                 ▼
      SQLite: data/db/bridge.sqlite
                 │
    ┌────────────┴────────────────────┐
    ▼                                 ▼
┌──────────────┐          ┌───────────────────────┐
│ scrapers/    │          │ src/agents/           │
│  ptt.py      │          │  base.py              │
│  news_rss.py │          │  issue_landscape.py   │
│  (fb via API)│          │  sentiment_extractor  │
└──────┬───────┘          │  strategic_dialectic  │
       │                  │  prompts/*.md         │
       ▼                  └──────────┬────────────┘
   raw posts                         │
       │                             ▼
       └──────► evidence_package ──► Agent 4 ──► 策略建議書
                                                 │
                                                 ▼
                                     reports/{trace,brief,dashboard}
```

### 5.1 Layering 原則（跟 district-analyst 同）
1. **抽取層（scrapers + SQL）**：絕不做推理，純搬運+去重+正規化
2. **推理層（agents）**：吃 evidence_package，不直接碰 raw data
3. **表達層（報告/dashboard）**：吃 agent output，不重做推理

若結論看起來怪 → 先判斷是抽取錯 (fix scraper/SQL) 還是推理錯 (fix prompt)。

### 5.2 Agent 清單

| Agent | 職責 | Prompt |
|-------|------|--------|
| `issue_landscape` | 事實地景：時間軸、爭議軸、利害關係人 | `prompts/issue_landscape.md` |
| `sentiment_extractor` | 對每條發言標註：主題、情緒、族群、一句推理 | `prompts/sentiment_extractor.md` |
| `strategic_dialectic` | 產出三個 framing、各自反證、推薦+為什麼 | `prompts/strategic_dialectic.md` |

> 注意 "scraper" 本身不是 agent — 它在抽取層，做純搬運。

## 6. 工作規則（給 Claude 自己看的）

### 6.1 事實查驗 — DO NOT HALLUCINATE
- 所有 **數字、日期、人名、政黨** 必須能追到 `data/facts/*.md` 或
  scraper 抓下來的 raw。
- 若資料裡沒有，**承認不知**、記下來，不要補。
- **2.5 公尺** 是事實錨 — 任何把它說成其他數字的 output 都是 bug。

### 6.2 政治敏感度
- 不要預設 NPP 會支持某個立場 — 讓辯證 agent 跑出來
- 不要把 陳世軒 當 NPP（他是 TPP）— 這種事實錯誤是送命
- 「黃國昌」在現階段是 **民眾黨總召**，不是 NPP 的人（2019 以後退出）

### 6.3 版本控制
- `data/facts/*.md` 改動要在檔尾註明 `Last updated: YYYY-MM-DD`
- Prompt 檔改動要保留舊版於 `prompts/archive/`
- 每次跑 agent 的 trace 存在 `reports/traces/<agent>_<timestamp>.md`

### 6.4 什麼時候停下來問人
- 發現 TPP/KMT 對此議題 **新聲明** → 先問，不要自己改 framing
- 發現官方數字跟文件矛盾 → 以官方新聞稿為準，標註 conflict
- 推理走進「建議 NPP 激烈反對通車」這種方向 → 停、問 — 這不是好的 posture

## 7. 交付物（deliverables）

目標是 3 件：

1. **`報告書.pdf`（主檔，20–30 頁）**
   - 摘要（1 頁）
   - Ch.1 議題地景
   - Ch.2 輿情結構
   - Ch.3 AI 設計思路（**這章最重要**）
   - Ch.4 NPP 切入機會（3 個 framing + 推薦）
   - Ch.5 下一步行動（短/中/長程）
   - 附錄：方法論、資料源、prompts

2. **`dashboard.html`（單檔自持）**
   - 輿情情緒時間曲線（2026-04-19 機車道爭議前後）
   - 主題聚類占比
   - 族群差異比較
   - 關鍵事件 annotation

3. **`repo.zip` 或 GitHub link**
   - README 講清架構
   - `prompts/` 可被非工程師編輯
   - `reports/traces/` — 每次 agent 執行的完整 evidence + prompt + response

加分項：
- **一頁「主張摘要」single-pager** — 內部轉傳用
- **一分鐘 walk-through 影片** — 錄跑 pipeline + 講設計哲學

## 10. 面試判分維度（給自己看的）

他們看這份報告時，心裡打分的是：

1. **政治敏感度** — 你懂不懂這個議題？你懂不懂 NPP？（fact check 要過關）
2. **AI 方法論** — 你的 agent 是資料管道還是推理單元？prompt 能不能被審查？
3. **決策可用性** — 下週 NPP 黨工能不能拿這份報告開會？
4. **重複性** — 這套方法換一個議題能不能再跑？（這是你跟 district-analyst 的連結賣點）

**不是看誰分析最全面 — 是看誰能把 AI 用在政治判斷上且不變玄學。**
