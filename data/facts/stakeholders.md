## 0. Schema 設計聲明（新）

### 0.1 為什麼是 9 個 code 而不是 5 個或 15 個

這 9 個 demographic code 是 **issue-specific scheme**，**不是** 套用既有社會科學
taxonomy（Ekman 六情緒、Hofstede 文化維度等）。設計原則四條：

1. **互斥 + 窮舉**：每則貼文只能歸一碼；無法歸的才落 unknown。
2. **對應利害關係人四象限**（見 §1）：地理型 × 功能型 × 議題型三軸涵蓋。
3. **從 PTT 短文可推斷**：每個 code 需有可驗證的語言訊號（例如 biker 有
   「我們騎車的」、resident_tamsui 有「我住淡水」、engineer_tech 有工程術語）。
   無明顯訊號的 code 不值得立。
4. **對 NPP 戰略有用**：每個 code 對應一個 NPP 需要知道的戰略訊號
   （誰支持、誰反對、誰沉默、誰可結盟）。

### 0.2 三軸分類

```
地理型（2）      resident_tamsui  — 淡水舊市區/新市鎮
                resident_bali    — 八里

功能型（4）      commuter         — 台北↔淡水/八里通勤族
                biker            — 機車族視角（我們騎車的、機車族）
                tourist          — 觀光客（夕陽、拍照、北海岸）
                real_estate      — 房仲/建商/投資視角

議題型（3）      env_concerned    — 環境/生態（河口、鳥類、光害）
                political_actor  — 自述為政治發言者
                engineer_tech    — 工程/技術專業
```

### 0.3 Schema 的三個自覺 limitation

**1. 這 9 個 code 是研究者主觀判斷，不是科學**
我可以增/減/重組 code，分析結果會變。例如若加 `bus_rider` 公車族 → 會從
`commuter` 拆出來。這是 **issue-specific** 分類，不是普適分類。
**若明天要做淡北道路分析，9 碼不原封不動複製；會重跑 stakeholder mapping，
產出該議題的 8 碼或 11 碼。Schema 是議題的函數，不是 agent 的常數。**

**2. 無法抽 intersection（交集身份）**
一個人可同時是 biker + resident_tamsui + commuter。但 schema 強制選一碼
（取最強訊號）。這會偏向把 biker 標為 biker（機車族語言最顯眼），
壓抑他的在地身份。這是 trade-off：犧牲多重身份換訊號精度。

**3. 自述 ≠ 實際**
`political_actor` 指「自述為政治發言者」。實務上多數人罵陳世凱但不會自稱
政治人物，會被標 biker 或 unknown。這會 **系統性低估政治化討論的真實比例**。

### 0.4 V2 追加：Layer 2 unknown 細分（2026-04-24）

v1 抽取 1030 筆中有 377 筆（37%）被標 unknown，比例偏高。不是 agent 失敗，
是 prompt 限制：當 9-code 訊號不足時，v1 選擇保守標 unknown。

v2 設計三層分類：
- **Layer 1**：原 9 code（v2 若有線索仍優先嘗試）
- **Layer 2**：7 個 sub-category — `sarcastic_noise`、`generic_critique`、
  `generic_support`、`pure_emotion`、`off_topic`、`short_fact`、`meta_discussion`
- **Layer 3**：真無法歸類的 `unknown`

v2 對 377 筆 unknown 重跑後，真 unknown 大幅降低，資料品質更透明可審查。
**v2 不是修正 v1 的錯，是延伸 v1 的 schema**。v1/v2 結果同時保存，dashboard
優先採用 v2（若存在），這讓整個 extraction 演化過程可被 replay。

### 0.5 對面試方的訊息

> **「這個 9-code schema 不是因為它科學，是因為它對此議題有用。
> 任何時候都可以被推翻——推翻它的標準是『新議題下有更適合的 schema』，
> 不是『存在一個更好的通用分類』。」**
